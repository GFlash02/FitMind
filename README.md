# 🦞 AI 个人助理 — 产品说明文档

**文档版本**：v1.0 | **更新时间**：2026-03-19 | **适配系统**：Windows/macOS/Linux（支持Python 3.8+环境）

## 一、产品定位

AI 个人助理是一款面向在校学生和职场人士的智能健康管理工具，聚焦“轻量、高效、个性化”核心，核心理念是：用 AI 替代繁琐的手动记录与分析，让用户只需“说一句话/拍一张照”，就能轻松完成运动记录、饮食分析、健康目标规划和数据总览，解放双手的同时，实现更科学的生活方式。

本产品区别于传统聊天机器人，将大语言模型（LLM）深度嵌入四大核心健康场景（运动、饮食、健康目标、数据总览），具备数据存储、进度追踪、智能分析、个性化建议四大核心能力，真正成为用户身边可依赖的“私人健康智能管家”，适配学生党、职场人日常健康管理需求。

## 二、解决的核心痛点

针对在校学生、职场人士日常健康管理中的高频困扰，对比传统方式的弊端，本产品给出精准解决方案：

| 核心痛点 | 传统方式弊端 | 本产品解决方案 |
|----------|--------------|----------------|
| 运动后不知消耗多少热量，无法精准把控 | 手动查表估算，受运动强度、个人体质影响，误差大，无法形成连续记录和分析 | AI 结合用户个人体重、运动类型、运动时长、强度，基于专业 MET（代谢当量）模型精准计算热量消耗，记录历史数据并支持趋势分析 |
| 饮食热量、营养成分查询费时费力 | 手动查询食物数据库，需逐一输入食材、分量，操作繁琐，耗时久，且无法精准匹配家常餐、外卖等场景 | 无需复杂操作，仅用文字描述一餐或上传餐食照片，AI 秒出热量、营养成分，适配家常餐、外卖、聚餐等多种场景 |
| 减肥/增肌计划难以坚持，效果不佳 | 网上搜索通用方案，未结合个人体质、生活习惯，针对性弱，易半途而废，无进度反馈 | 基于用户真实数据生成专属计划，实时同步进度，根据体重变化、执行情况动态调整计划，降低坚持难度 |
| 健康数据分散，无法掌握整体状态 | 运动、饮食、体重等数据分散在不同记录工具，无法汇总分析，看不到整体健康趋势 | 统一数据总览界面，整合所有健康数据，通过可视化图表直观展示，让用户快速掌握自身健康状态 |

## 三、功能详解

### 💪 功能一：运动记录（核心健康场景）
**使用场景**：适用于用户日常运动（室内健身、户外跑步、跳绳、瑜伽等）记录，快速了解单次运动热量消耗，追踪长期运动习惯，辅助达成健康目标（减脂/增肌），适配学生党课余运动、职场人碎片化健身等场景。

**操作流程**：
1. 通过飞书客户端与 openclaw 机器人交互（支持私聊、群聊触发，无需额外下载APP）；
2. 自然语言输入运动相关信息（如“今天晚上跑步30分钟，速度6km/h，体重65kg”），或上传运动截图；
3. AI 自动解析运动数据，精准计算热量消耗，同步计入后台分析数据库，生成单次运动报告。

**AI 核心能力**：
- 精准热量计算：基于 MET 专业模型，误差控制在±5%以内；
- 个性化运动建议：根据用户运动类型、历史记录给出针对性建议；
- 历史记录统计：支持按日/周/月统计运动数据，生成运动趋势图表。

**补充说明**：支持有氧运动、力量训练、柔性运动等全类型运动记录，可手动补充特殊运动类型及参数。

### 🍱 功能二：饮食分析（核心健康场景）
**使用场景**：适用于用户日常饮食管理，快速了解每餐的热量、营养成分，判断饮食是否均衡，辅助减脂、增肌或健康饮食目标，适配家常餐、外卖、聚餐、校园餐等多种饮食场景。

**操作流程**：
1. 通过飞书客户端与 openclaw 机器人交互，支持私聊、群聊触发；
2. 提供餐食相关信息：文字描述（如“早餐：1个鸡蛋、1杯豆浆、1个包子”）或上传餐食照片；
3. AI 快速解析餐食数据，生成详细饮食分析报告，同步计入数据库，支持历史查询。

**AI 输出内容**：
- 基础数据：总热量（kcal）、蛋白质/碳水/脂肪含量（克）；
- 营养评价：按营养素分配比例给出“优/良/一般/较差”评价，指出不足；
- 个性化饮食建议：结合用户健康目标、历史饮食记录给出针对性建议；
- 历史记录：支持按日/周/月查询饮食记录，汇总摄入数据。

**联动功能**：自动计算“今日热量收支”（摄入-消耗=净摄入），红色/绿色标注是否超标，直观判断热量是否符合目标。

### 🎯 功能三：健康目标（核心规划场景）
**使用场景**：适用于用户设定减脂、增肌、维持体重等健康目标，获取 AI 定制的个性化健康计划，实时追踪目标进度，避免盲目规划。

**操作流程**：
1. 填写个人基础信息及目标参数（当前体重、目标体重、身高、年龄、性别、目标周期、日常活动水平）；
2. 确认保存后，AI 基于 TDEE（每日总能量消耗）模型生成专属健康计划；
3. 在“数据总览”界面查看目标进度，系统根据日常记录动态调整计划。

**AI 输出内容**：
- 每日推荐热量摄入范围；
- 三大营养素分配比例及具体克数；
- 每周运动计划（类型、时长、频率、重点）；
- 3~5条简单易执行的饮食原则；
- 针对性注意事项。

**数据可视化**：体重趋势折线图展示体重变化，目标体重虚线标注，进度百分比实时更新。

### 📊 功能四：数据总览（核心汇总场景）
**使用场景**：适用于用户每日/每周/每月查看健康数据全貌，快速掌握饮食、运动、体重及目标进度，进行复盘调整。

#### 核心展示指标
| 核心指标 | 具体说明 |
|----------|----------|
| 今日摄入热量 | 当天所有餐次热量合计，对比每日推荐值，标注是否超标 |
| 今日运动消耗 | 当天所有运动热量消耗合计，标注运动时长、累计次数 |
| 净摄入热量 | 今日摄入 - 今日运动消耗，直观判断热量盈亏 |
| 距目标体重 | 当前体重与目标体重的差距，标注目标达成百分比 |

#### 可视化图表（交互式，支持放大、筛选）
- 近7天热量收支对比柱状图；
- 近30天运动类型分布饼图；
- 近30/90天体重趋势折线图；
- 近7天营养成分趋势图。

### 📋 附加功能：任务规划
**使用场景**：适用于学生/职场人士规划学业/工作任务，解决任务繁杂、拖延、遗漏问题。

**操作流程**：
1. 自然语言输入任务（如“下周一之前完成项目方案撰写，需要调研3个竞品”）；
2. AI 自动分解为带优先级、时间估算、截止时间的子任务；
3. 用户可调整子任务，完成后标记“已完成”，系统追踪进度。

**AI 核心能力**：智能任务分解、优先级排序、进度提醒、完成率统计。

### 📅 附加功能：日程管理
**使用场景**：适用于记录日常日程，实现提醒、高效管理，避免遗漏重要事项。

**操作流程**：
1. 自然语言输入日程信息（如“明天下午3点在公司会议室开项目推进会，记得带方案和笔记本”）；
2. AI 自动解析时间、地点、核心事项、备注，生成标准化日程；
3. 系统在日程开始前30分钟（可调整）发送飞书提醒。

**核心特性**：自然语言解析、联动任务规划、关键词查询、重复日程设置。

## 四、技术架构

本产品采用“轻量部署、本地优先、高兼容性”的技术架构，无需专业技术能力，开箱即用，同时保障数据隐私安全。

### 核心技术栈
| 技术模块 | 具体技术 | 核心作用 |
|----------|----------|----------|
| 前端框架 | Streamlit（Python Web UI） | 快速构建轻量美观的Web界面，支持交互式图表，操作简单 |
| AI 引擎 | OpenAI 兼容 API | 支持GPT-4o、Claude等模型，实现自然语言解析、智能计算 |
| 数据存储 | SQLite 本地数据库 | 本地存储数据，无需服务器，保障隐私 |
| 可视化 | Plotly（交互式图表） | 生成折线图、柱状图、饼图，支持放大/筛选/查看详情 |
| 数据处理 | Pandas | 高效处理健康数据，实现统计分析与联动计算 |
| 交互入口 | 飞书机器人（openclaw） | 便捷的自然语言交互，支持私聊/群聊触发 |
| 视觉识别 | AI 视觉理解模型 | 解析餐食照片，识别食材及分量 |

### 数据表结构（本地 SQLite）
所有数据本地存储，不上传个人信息，表结构如下：
- `exercise_logs`（运动记录）：运动ID、时间、类型、时长、强度、体重、热量消耗、AI建议等；
- `diet_logs`（饮食记录）：饮食ID、时间、餐次、描述/照片、热量、三大营养素、评价、AI建议等；
- `weight_logs`（体重记录）：体重ID、时间、数值、变化量等；
- `health_goals`（健康目标）：目标ID、个人信息、体重数据、目标周期、推荐热量、营养素比例等。

### 隐私安全说明
1. 数据本地存储：所有健康数据均存储在本地，完全由用户掌控；
2. 无账号注册：无需注册账号、绑定手机号，避免信息泄露；
3. 数据可导出/删除：支持导出CSV格式，或删除数据库文件彻底清除数据。

## 五、快速启动

### 环境要求
- Python 版本：3.8 及以上（推荐 3.10）；
- 依赖库：详见 requirements.txt（自动安装）；
- 网络环境：启动时需联网（安装依赖、调用AI API），后续可离线使用（仅查看历史数据）；
- 飞书客户端：需安装（用于交互和接收提醒）。

### 启动步骤（命令行操作）
```bash
# 1. 进入源码目录
cd AI-Personal-Assistant

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置API（复制示例配置文件并编辑）
cp .env.example .env

# 4. 启动应用
streamlit run app.py
