一、产品定位

AI 个人助理是一款面向在校学生和职场人士的智能健康管理工具，聚焦“轻量、高效、个性化”核心，核心理念是：用 AI 替代繁琐的手动记录与分析，让用户只需“说一句话/拍一张照”，就能轻松完成运动记录、饮食分析、健康目标规划和数据总览，解放双手的同时，实现更科学的生活方式。

本产品区别于传统聊天机器人，将大语言模型（LLM）深度嵌入四大核心健康场景（运动、饮食、健康目标、数据总览），具备数据存储、进度追踪、智能分析、个性化建议四大核心能力，真正成为用户身边可依赖的“私人健康智能管家”，适配学生党、职场人日常健康管理需求。

二、解决的核心痛点

针对在校学生、职场人士日常健康管理中的高频困扰，对比传统方式的弊端，本产品给出精准解决方案，具体如下：

核心痛点

传统方式弊端

本产品解决方案

运动后不知消耗多少热量，无法精准把控

手动查表估算，受运动强度、个人体质影响，误差大，无法形成连续记录和分析

AI 结合用户个人体重、运动类型（跑步/跳绳/健身等）、运动时长、强度，基于专业 MET（代谢当量）模型精准计算热量消耗，记录历史数据并支持趋势分析

饮食热量、营养成分查询费时费力

手动查询食物数据库，需逐一输入食材、分量，操作繁琐，耗时久，且无法精准匹配家常餐、外卖等场景

无需复杂操作，仅用文字描述一餐（如“中午吃了1碗米饭、1份番茄炒蛋、1杯牛奶”）或上传餐食照片，AI 秒出热量、营养成分，适配家常餐、外卖、聚餐等多种场景

减肥/增肌计划难以坚持，效果不佳

健康数据分散，无法掌握整体状态

运动、饮食、体重等数据分散在不同记录工具（备忘录、表格、运动APP），无法汇总分析，看不到整体健康趋势

统一数据总览界面，整合运动、饮食、体重、目标等所有数据，通过可视化图表直观展示，让用户快速掌握自身健康状态

三、功能详解

💪 功能一：运动记录（核心健康场景）

使用场景：适用于用户日常运动（室内健身、户外跑步、跳绳、瑜伽等）记录，快速了解单次运动热量消耗，追踪长期运动习惯，辅助达成健康目标（减脂/增肌），适配学生党课余运动、职场人碎片化健身等场景。

操作流程：

- 通过飞书客户端与 openclaw 机器人交互（支持私聊、群聊触发，无需额外下载APP）；

- 自然语言输入运动相关信息（如“今天晚上跑步30分钟，速度6km/h，体重65kg”），或上传运动截图（如运动APP记录）；

- AI 自动解析运动数据，精准计算热量消耗，同步计入后台分析数据库，生成单次运动报告。

AI 核心能力：

- 精准热量计算：基于 MET（代谢当量）专业模型，结合用户个人体重、运动类型、运动时长、强度，精准估算单次运动热量消耗，误差控制在±5%以内；

- 个性化运动建议：根据用户运动类型、历史运动记录，给出针对性建议（如“长期跑步建议搭配拉伸运动，避免小腿肌肉堆积”“减脂期间建议每周增加2次力量训练，提高基础代谢”）；

- 历史记录统计：支持按日/周/月统计运动数据，包括累计运动次数、总运动时长、总消耗热量，生成运动趋势图表，直观展示运动习惯变化。

补充说明：支持运动类型全覆盖，包括有氧运动（跑步、跳绳、游泳等）、力量训练（举铁、俯卧撑等）、柔性运动（瑜伽、普拉提等），可手动补充特殊运动类型及参数。

🍱 功能二：饮食分析（核心健康场景）

使用场景：适用于用户日常饮食管理，快速了解每餐的热量、营养成分，判断饮食是否均衡，辅助减脂、增肌或健康饮食目标，适配家常餐、外卖、聚餐、校园餐等多种饮食场景。

操作流程：

1. 通过飞书客户端与 openclaw 机器人交互，支持私聊、群聊触发；

2. 提供餐食相关信息：可文字描述（如“早餐：1个鸡蛋、1杯豆浆、1个包子”），或上传餐食照片（清晰拍摄餐食全貌，AI 通过视觉理解模型识别食材及分量）；

3. AI 快速解析餐食数据，生成详细饮食分析报告，同步计入数据库，支持历史查询。

AI 输出内容：

- 基础数据：精准输出餐食总热量（单位：kcal），蛋白质、碳水化合物、脂肪的具体含量（单位：克）；

- 营养评价：根据三大营养素分配比例、热量合理性，给出“优/良/一般/较差”的评价，明确指出不足（如“蛋白质摄入不足，建议增加鸡蛋、瘦肉摄入”）；

- 个性化饮食建议：结合用户健康目标（减脂/增肌/维持健康）、历史饮食记录，给出针对性建议（如“减脂期间，晚餐建议减少碳水摄入，可替换为杂粮饭”）；

- 历史记录：支持按日/周/月查询饮食记录，汇总每日/每周摄入热量、营养成分，方便用户复盘饮食情况。

联动功能：

若当天已有运动记录，系统会自动计算并显示“今日热量收支”（摄入热量 - 运动消耗热量 = 净摄入热量），用红色/绿色标注（红色表示热量超标，绿色表示热量合理），帮助用户直观判断当天热量是否符合目标要求，避免盲目饮食。

🎯 功能三：健康目标（核心规划场景）

使用场景：适用于用户设定减脂、增肌、维持体重等健康目标，获取 AI 定制的个性化健康计划，实时追踪目标进度，避免盲目规划，适配有明确健康需求的用户。

操作流程：

1. 在产品界面填写个人基础信息及目标参数，包括：当前体重、目标体重、身高、年龄、性别、目标周期（如“3个月减脂10斤”）、日常活动水平（久坐/轻度活动/中度活动/高强度活动）；

2. 确认信息并保存，AI 基于 TDEE（每日总能量消耗）模型，结合个人数据生成专属健康计划；

3. 用户可在“数据总览”界面查看目标进度，系统会根据用户每日饮食、运动记录，定期调整计划（如体重下降过快，适当增加热量摄入）。

AI 输出内容：

- 每日推荐热量摄入：基于 TDEE 精准计算，结合目标（减脂/增肌）给出合理的每日热量摄入范围，避免热量过高或过低；

- 三大营养素分配比例：明确每日蛋白质、碳水化合物、脂肪的摄入比例及具体克数，适配不同目标（如减脂期高蛋白、低GI碳水，增肌期高碳水、高蛋白）；

- 每周运动计划：结合用户活动水平、目标，制定具体的运动类型、时长、频率（如“每周3次有氧运动，每次30分钟；2次力量训练，每次40分钟”），标注运动重点（如减脂重点练有氧运动，增肌重点练力量训练）；

- 饮食原则：给出3~5条简单易执行的饮食原则（如“多喝水，每日饮水量不低于1500ml”“避免高油高糖食物，优先选择天然食材”）；

- 注意事项：结合用户个人情况，给出针对性注意事项（如“久坐人群，每小时起身活动5分钟”“减脂期间避免过度节食，防止代谢下降”）。

数据可视化：

体重趋势折线图，清晰展示每日/每周体重变化，目标体重用虚线标注，进度百分比实时更新，让用户直观看到目标达成情况，增强坚持动力。

📊 功能四：数据总览（核心汇总场景）

使用场景：适用于用户每日、每周或每月查看健康数据全貌，快速掌握自身饮食、运动、体重及目标进度，进行复盘调整，适配需要定期总结健康状态的用户。

- 体重趋势折线图：展示近30天/90天体重变化，标注目标体重虚线，支持查看每日体重详情，便于复盘调整；

- 营养成分趋势图：展示近7天蛋白质、碳水化合物、脂肪的摄入变化，帮助用户优化饮食结构，确保营养均衡。

使用场景：适用于在校学生规划学业任务（作业、考试复习）、职场人士规划工作任务（项目推进、会议准备），解决任务繁杂、拖延、遗漏等问题，提升效率。

操作流程：

- 自然语言输入任务（如“下周一之前完成项目方案撰写，需要调研3个竞品”）；

- AI 自动分解任务，生成带优先级（高/中/低）、时间估算（分钟级）、截止时间的子任务；

- 用户可手动调整子任务优先级、截止时间，完成后标记“已完成”，系统实时追踪进度。

AI 核心能力：

- 智能任务分解：将复杂任务拆解为可落地的子任务（如“项目方案撰写”拆解为“调研竞品、搭建框架、撰写初稿、修改定稿”）；

- 优先级排序：根据任务截止时间、重要程度，自动标注优先级，提醒用户优先处理高优先级任务；

- 进度提醒：临近截止时间的任务，系统通过飞书消息主动提醒，避免遗漏；

- 进度统计：支持按日/周/月统计任务完成率，展示未完成、已完成任务数量，帮助用户复盘效率。

使用场景：适用于用户记录日常日程（会议、上课、约会、重要事项），实现日程提醒、高效管理，避免遗漏重要事项，适配学生党、职场人士的日常规划需求。

操作流程：

- 自然语言输入日程信息（如“明天下午3点在公司会议室开项目推进会，记得带方案和笔记本”）；

- AI 自动解析日程的时间、地点、核心事项、备注信息，生成标准化日程；

- 系统在日程开始前30分钟（可手动调整提醒时间），通过飞书消息发送提醒，确保用户不会遗漏。

核心特性：

- 自然语言解析：无需手动填写表单，一句话即可完成日程录入，支持模糊时间（如“下周一下午”“明天上午”）解析；

- 联动任务规划：若日程与任务相关（如“项目推进会”对应“准备会议方案”任务），系统自动关联任务，提醒用户提前完成相关任务；

- 日程查询：支持按日期、关键词查询日程，方便快速找到相关安排；

- 重复日程：支持设置重复日程（如“每周一早上9点开会”），无需重复录入，节省操作时间。

数据表结构（本地 SQLite）

所有数据表均存储在本地，不上传任何个人信息，保障隐私安全，具体表结构如下：

- exercise_logs（运动记录）：存储运动ID、运动时间、运动类型、时长、强度、体重、热量消耗、AI建议等信息；

- diet_logs（饮食记录）：存储饮食ID、饮食时间、餐次（早/中/晚/加餐）、餐食描述/照片、热量、蛋白质、碳水、脂肪、营养评价、AI建议等信息；

- weight_logs（体重记录）：存储体重ID、记录时间、体重数值、变化量等信息；

- health_goals（健康目标）：存储目标ID、用户基础信息（身高、年龄、性别）、当前体重、目标体重、目标周期、每日推荐热量、营养素比例等信息；

隐私安全说明

1. 数据本地存储：所有用户数据（饮食、运动、体重、健康目标）均存储在本地 SQLite 数据库，不上传至任何服务器，完全由用户自主掌控；

2. 无账号注册：无需注册账号、绑定手机号，避免个人信息泄露；

3. 数据可导出/删除：支持将所有数据导出为CSV格式，也可手动删除数据库文件，彻底清除所有数据。

五、快速启动

本产品部署简单，无需专业技术能力，只需按照以下步骤操作，即可快速启动使用，适配 Windows、macOS、Linux 系统（需提前安装 Python 3.8+）。

环境要求

- Python 版本：3.8 及以上（推荐 3.10 版本，兼容性更好）；

- 依赖库：详见 requirements.txt（自动安装，无需手动下载）；

- 网络环境：启动时需联网（用于安装依赖、调用 AI API），后续本地使用可离线（离线状态下无法调用 AI 能力，仅可查看历史数据）；

- 飞书客户端：需安装飞书（用于与 openclaw 机器人交互，接收提醒）。

启动步骤（命令行操作）

# 1. 下载产品源码包，解压后进入源码目录（命令行切换路径）
cd AI-Personal-Assistant

# 2. 安装依赖（自动安装所有所需依赖，耐心等待）
pip install -r requirements.txt

# 3. 配置 API（复制示例配置文件，编辑填写个人 API 密钥）
cp .env.example .env

# 4. 启动应用（启动 Streamlit 服务）
streamlit run app.py

访问使用

启动成功后，命令行会提示访问地址（默认：http://localhost:8501），打开浏览器，输入该地址即可进入产品界面，完成个人基础信息设置后，即可开始使用所有功能。

API 配置说明（.env 文件）

打开解压后的源码目录，找到 .env 文件（复制 .env.example 生成），用记事本或代码编辑器打开，填写以下参数：

# OpenAI 兼容 API 配置
OPENAI_API_KEY=你的API密钥（必填）
OPENAI_BASE_URL=https://api.openai.com/v1（可替换为国内兼容接口）
OPENAI_MODEL=gpt-4o（可替换为其他兼容模型，如 claude-3-opus、qwen-max 等）

补充说明：

- API 密钥获取：需前往对应 AI 模型平台（如 OpenAI、国内 AI 服务商）注册账号，获取 API 密钥；

- 国内模型适配：支持替换为国内 OpenAI 兼容接口（如智谱、通义千问等），只需修改 OPENAI_BASE_URL 为对应接口地址，确保模型兼容即可；

- API 费用：使用 AI 能力（自然语言解析、热量计算、计划生成等）会消耗 API 额度，具体费用根据所选模型、使用频率而定，建议根据自身需求选择合适的模型。

常见问题（启动失败排查）

- 问题1：pip install 安装依赖失败 → 解决方案：升级 pip 版本（pip install --upgrade pip），或手动安装失败的依赖（pip install 依赖名称）；

- 问题2：启动后浏览器无法访问 → 解决方案：检查 Python 版本是否符合要求，关闭占用 8501 端口的程序，重新执行 streamlit run app.py；

- 问题3：AI 功能无法使用 → 解决方案：检查 .env 文件中 API 密钥、接口地址是否填写正确，确保网络通畅，API 密钥未过期。

六、产品亮点总结

本产品聚焦用户健康管理核心需求，结合 AI 技术打造轻量、高效、个性化的智能健康助理，相比同类产品，核心亮点如下：

1. 零门槛交互 — 全程支持自然语言输入、照片上传，无需学习任何操作规范，无论是记录运动、饮食，还是规划健康目标，一句话/一张照即可完成，适配所有用户群体，尤其适合不熟悉复杂操作的用户。

2. AI 深度健康计算 — 区别于传统“展示型”工具，AI 真正参与热量计算、营养分析、计划生成，从 MET 代谢当量估算、TDEE 热量计算，到个性化运动/饮食建议、计划动态调整，全程智能化，无需用户手动干预。

3. 数据闭环联动 — 运动、饮食、体重、健康目标四大场景数据互通，形成完整闭环（如饮食+运动联动计算热量收支），让用户无需在多个工具间切换，高效管理日常健康。

4. 极致个性化 — 所有建议、计划均基于用户真实数据（体重、年龄、活动量、目标等）生成，非通用模板，适配不同用户的个性化需求（如学生党减脂、职场人增肌、久坐人群健康管理）。

5. 本地优先，隐私安全 — 采用 SQLite 本地数据库，所有数据不上传、不泄露，无需注册账号，用户完全掌控个人健康数据，解决隐私泄露顾虑，尤其适合关注健康数据隐私的用户。

6. 轻量部署，开箱即用 — 单文件应用，无需复杂配置，只需执行3步命令即可启动，适配多种操作系统，无需专业技术能力，个人用户可快速部署使用。

7. 健康场景全覆盖 — 整合运动记录、饮食分析、健康目标、数据总览四大核心健康场景，一站式解决用户日常健康管理的所有痛点，无需下载多个工具。
